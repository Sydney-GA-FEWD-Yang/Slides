CSS Box Model

Using CSS


            div{
    margin: 10px 20px 30px 40px;
    border: 5px solid black;
    padding: 40px 30px 20px 10px;
}

Margin

In the above, we used 4 different pixels in margin. These represents top right bottom left


            div{
    margin: top right bottom left;
}

But margin can also accept less than 4 inputs

Search for margin in google

Targetting specific margin side


            div{
    margin-top: 10px;
    margin-right: 20px;
    margin-bottom: 30px;
    margin-left: 40px;
}

This is equivalent to the previous margin statement

Border

Border accepts up to 3 inputs, border-width, border-style, border-color


            p{
    border: 2px dotted blue;
}

This is equivalent to


            p{
    border-width: 2px;
    border-style: dotted;
    border-colour: blue;
}

Did you spot the mistake?

How do I know all the border-styles?

Padding

Syntax is the same as margin, just replace margin with padding


            div{
    padding: 10px 20px 30px 40px;
}

Equivalent to


            div{
    padding-top: 10px;
    padding-right: 20px;
    padding-bottom: 30px;
    padding-left: 40px;
}

DOM Tree

DOM - document object model

It's the same as the html elements in our page

HTML is a tree data structure

Draw this dom tree


            <html>
    <head>
        <title>My Work</title>
        <meta charset="utf-8"/>
        <meta name="author" content="Yang Li"/>
    </head>
    <body>
        <h1>My Work</h1>
        <p>This is my <a>work</a></p>
        <ul>
            <li>IBM</li>
            <li>Nextgen.net</li>
            <li>Bauer Media</li>
        </ul>
    </body>
</html>

Terminology

  • li is a child of ul
  • body is the parent p
  • body is a sibling of head
  • li is a descendant of body

CSS Selectors

Review - We've learnt how to select tags


            div {
    color: blue;
}

More specific?

We can select more specific tags as such


            body > div {
    color: red;
}

the greater than symbol (>) means child

In our case, we are selecting all divs that are childrens of body


            div {
    color: blue;
}

body > div {
    color: red;
}
        

            <body>
    <div>Hello world</div>
    <div>
        <div>Which colour will I be?</div>
    </div>
</body>

Selecting descendants


            body div {
    color: red;
}

The rule above will colour all divs that are descendants of body RED


div {
    color: blue;
}

body div {
    color: red;
}
        

<body>
    <div>Hello world</div>
    <div>
        <div>Which colour will I be?</div>
    </div>
</body>

Classes and Ids

More CSS selectors

.Classes


            <div class="my-class">
    content
<div>

To style this in css


            .my-class{
    color: red;
}

Notice the "." infront of ".my-class"

#Ids


            <div id="my-id">
    content
<div>

To style this in css


            #my-class{
    color: red;
}

Notice the "#" infront of "#my-class"

Classes vs Ids

Classes Ids
Use classes to style an element or group of element Use Ids to target unique and individual elements
Use classes when in doubt Use Id's when you want a linkable section

You can use ids to create section links

Example : https://en.wikipedia.org/wiki/HTML#HTTP
